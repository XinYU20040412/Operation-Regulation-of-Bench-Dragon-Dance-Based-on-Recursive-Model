# 1-paper 目录说明

本目录用于管理论文正文、赛题原文和答辩材料，是项目对外呈现的核心文档区。

## 目录职责

- 保存最终提交版本论文与题目原文，确保“结论可追溯”。
- 维护论文图表与代码结果的一致性，避免出现“图表与程序输出不匹配”。
- 作为答辩材料的索引入口，便于快速定位关键图和关键结论。

## 当前文件

- `24年数模国赛论文.pdf`：论文主文档（提交版）。
- `A题.pdf`：赛题原文或官方题目说明。

## 建议的论文-代码对应关系

- 建模流程图、算法说明：对应 `../2-code/正式代码/README.md`。
- 关键结果图：优先使用 `../2-code/正式代码/viz_output/` 中自动生成图像。
- 数据表格（如逐帧指标）：由 `collision_metrics.csv` 二次整理得到。
- 关键结论（首次碰撞时刻、碰撞对）：以 `summary.json` 为准。

## 更新流程（推荐）

1. 先运行正式脚本，刷新 `viz_output` 产物。
2. 再更新论文中的图、表、结论文字。
3. 最后做一次“数值一致性检查”：论文中的时间、碰撞对、扫描参数需与 `summary.json` 一致。

## 协作注意事项

- 建议在文件名中标注版本或日期，避免多人协作时误覆盖。
- 若同一章节有多版稿件，建议新增子目录（如 `draft/`、`final/`）分层管理。
- 若代码参数有调整（扫描区间、步长等），必须同步记录到论文“实验设置”章节。
