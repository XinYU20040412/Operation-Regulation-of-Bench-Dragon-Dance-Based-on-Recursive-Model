# script 目录说明

本目录用于保存中间计算脚本、分题脚本和过程验证脚本，可视作“建模推导与试算工作区”。

## 目录定位

- 适合快速验证局部公式、局部参数和局部图像。
- 适合按题号拆分处理复杂任务，降低单脚本复杂度。
- 不直接作为最终提交结论来源。

## 文件用途（按常见用途）

- `求解龙头位置.py`：龙头轨迹与把手位置基础求解。
- `prob1.py`：问题一数值流程草稿与验证。
- `prob4_calc(1).py`、`prob4_head_positions2.py`：问题四相关计算过程脚本。
- `动静态结果对比图像.py`：动静态模型对比绘图。
- `第三问不稳定代码.py`、`问题五草稿.py`、`untitled3.py`：历史草稿或试验代码。

## 建议的使用方式

1. 在本目录完成思路验证和参数探索。
2. 对通过验证的逻辑进行函数化整理。
3. 迁移到 `../正式代码/` 并统一输入输出格式。

## 迁移到正式代码前的检查项

- 是否存在硬编码参数（应改为可配置参数）。
- 是否有清晰的输入输出定义。
- 是否可在同样参数下稳定复现结果。
- 是否补充了必要注释，便于团队成员接手。

## 注意事项

- 本目录脚本命名历史较长，存在草稿文件，运行前建议先阅读源码确认用途。
- 若脚本结果被用于论文，建议在提交前把核心逻辑同步到 `正式代码`。
